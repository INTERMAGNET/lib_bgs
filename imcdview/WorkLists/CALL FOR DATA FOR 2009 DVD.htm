This year the “call for data” is earlier because of VERY SERIOUS CHANGES REGARDING INTERMAGNET FORMATS:
1) New format for the Archival data on DVD for 2009 data (and onwards) – IAF V2.0
2) New Baseline File Format for 2009 data (and onwards) – IBF V2.00
You can find detailed information concerning new as well as old formats on
http://www.intermagnet.org/FormatData_e.html
 
The formal deadline for 2009 Definitive 1-min data is August 1, 2010.
 
The updated version of Java browser IMCDVIEW handling new formats will be available before long (March/April), a special email informing about where this software will be available will be sent to you.

As in previous years there are two steps of data collection. First, IMOs put their data on Paris server to so called "unchecked" directory (step1). After that the responsible persons for subgroups of IMOs check data and eventually correspond with IMOs and ask to correct data and so on. Then the responsible persons copy these data to "checked" directory (step2). The "checked" directory is not visible for IMOs. The presence of files in "checked" directory means that data have been checked by the responsible person.
NOTE: As you know in some cases the responsible person for observatory and subgroup is the same. In such cases, data should be placed only in "checked" directory. Please find the details at the end of this email.
 
Finally we would like to ask you to control all files destined for the DVD with regard to coherence of information included to different files (for example coordinates, elevation, addresses, instrumentation and so on).
Furthermore INTERMAGNET encourages IMOs to generate yearbooks, with the option to leave out the data sections. Such yearbooks contain valuable information describing observatory data (metadata). The yearbooks can be published on INTERMAGNET web http://www.intermagnet.org/Yearbooks_e.html.
 
If you need further information, please don't hesitate to ask.

===== REQUIRED FILES & METHODOLOGY OF CONTROL ===========
 
This year (2009 definitive data) each observatory should deliver the following data and information:
 
A. 12 CD-ROM 1 min binary data files, orientation XYZG (format IAFV2.0).
B. 1 baseline file (format IBFV2.00).
C. 1 readme-file.
D. 1 annual means file.
E. Information about readme-country file (Please state what you
     want: new file, changes or reuse of 2008-file?).
F. Information about About-screen for country.(Please state what
     you want: new-file, changes or reuse of 2008-file?)
G. Information about Map of country. (Please state what you
     want: new-file, changes or reuse of 2008-file?)
 
Please remember to send the information about changes or no changes
of the country file and screenplots. The DVD cannot be finished
before we have received all information.
We encourage you to add annual mean values of time intervals before 1980.
So if possible please include the complete time series of the observatory spanning back to its beginning.
Missing elements in an annual mean record should be represented with 9s.
Missing angular values should be coded as three 9 digits, a space, two 9 digits a dot and one 9 digit:
999 99.9
Missing field strengths should be coded as six 9 digits:
999999
 
 
HOW DO YOU CHECK YOUR DATA?
 
To check your 2009 data you should create a directory structure similar to the structure on the Intermagnet DVD,
this will allow you to use the imcdview.jar software. Please remember that ctry_inf and 2009maps subdirectories are necessary (even if empty).
 
The complete Intermagnet data includes the following files:
1.    x:\mag2009\sss\sss09mmm.bin            12 binary files
2.    x:\mag2009\sss\readme.sss                  1 readme file
3.    x:\mag2009\sss\sss2009.blv                   1 baseline file
4.    x:\mag2009\sss\yearmean.sss               1 annual means file
5.    x:\mag2009\maps2009\COUNTRY_ID.png   1 map for each country
6.    x:\mag2009\ctry_inf\readme.COUNTRY_ID  1 readme-file for  each country
7.    x:\mag2009\ctry_inf\COUNTRY_ID+srn.png  1 about-screen for each country
(sss is station ID, mmm is month)
The above structure is also required by the most of testing programs.
 
Before placing your data on Paris server please check your data
carefully. Intermagnet will make before long the updated Java browser for control.
 
We suggest to make at least the following tests before placing your
Intermagnet files on Paris:
1) Review binary data
    Software: Java bowser IMCDVIEW
2) Headers in binary data and discrepancies between XYZ and F
    Software: Java browser IMCDVIEW or imagchc5.exe (imagchc5.exe will be available on Paris ftp server)
    Please remember that D-conversion factor for XYZF orientation should be exactly 10000.
    NOTE: imagchc5.exe also will check, whether recent yearly mean values are included to yearmean.sss
    and whether the H annual mean value of the first line of the baseline file are consistent with 12 binary files.
3) Control of yearmean file
    Software: ymchk.exe (32bits console application)
4) K-numbers
    Software: Java browser IMCDVIEW
    Remark: Program Kasm.exe available on INTERMAGNET WEB makes possible
    to include K-numbers to binary files generated according Adaptative Smoothed method
5) BLV file
   Software: Java browser IMCDVIEW
